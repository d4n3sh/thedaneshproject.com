How to build a local DNS caching server

Being in Malaysia we are gifted with superior Internet speeds. NOT!! Services like openDNS are awesome but the lag between us and them often results in sluggish performance anyways. One way to improve performance is to use local DNS servers. I don’t use Streamyx’s DNS servers because they SUCK!!. TIME’s DNS servers are ok but I still prefer openDNS. To improve performance, I put together a local DNS caching-only server that forwards to openDNS. Now I have openDNS with lighting fast response. Let’s walk though the steps to get your own local DNS caching-only server setup. I’m using openSUSE 11 so the steps might vary depending on your distro. Install BIND pandora:~ # zypper in bind Edit /etc/named.conf pandora:~ # vi /etc/named..conf Uncomment the forwarders section. Update the default values with the values below. forwarders { 208.67.222.222; 208.67.220.220; }; <br /> forward only; Add the line ” forward only; ” This will tell BIND to only forward to the forwarders and not the ROOT servers. Start the service.
